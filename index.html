Introducción

Reto Html & CSS

Para aprender, prácticar y poner a prueba las habilidades en HTML y CSS. Que mejor manera que realizar un sitio web, que refleje los conocimientos y habilidades que actualmente se encuentra estudiando.

Será un reto desde el principio, escoger el tema, buscar los recursos, definir el contenido, optimizar cada paso, hacer que se vea bien y que todo funcione perfectamente.

Esperamos que todos puedan alcanzar su objetivo de creación del primero de muchos sitios web, desde hoy emprendemos el desafío de lograr este primer reto, un producto profesional para agregar a su portafolio como entregable del diplomado de fullstack.

¿Qué hay que hacer?

Enunciado
Busca

El primer paso, es tener un punto de partida que sirva de inspiración, por tanto es importante tener una fase de búsqueda.
Hobbies, pasatiempos, gustos musicales, ideas de negocio, portafolio...

Prototipa

Cuando se encuetra la inspiración, se debe pasar a planear como se creará el proyecto, en esta fase hay muchas herramientas que permiten acercarnos al resultado final.

Codifica

Luego de tener clara la idea en el prototipo, se pasa a la creación del proyecto, para esto es necesario escribir código HTML y CSS.

Prueba

Comparte el proyecto de manera pública entre tus compañeros y amigos, valida el funcionamiento en varios navegadores y dispositivos.

Publica

Finalmente deja una versión públicada en línea, para que cualquier persona tenga acceso y tenga acceso al contenido creado.

¿Qué debo entregar?

Criterios

Sitio web de una sola página, con un diseño responsivo, que muestre información de un tema de interés personal.

Estructura

Control de versiones en el repositorio, mínimo 20 commits. El readme debe estar personalizado.

Repositorio

Optimización de contenido, imágenes, íconos, enlaces, contenido externo. Todo debe verse bien y funcionar adecuadamente.

Rendimiento

La hoja de estilos debe estar separada del html, ninguna etiqueta html debe tener reglas CSS, la comunicación entre el HTML y el CSS debe ser a través de selectores.

Estética

Las etiquetas HTML, deben ser semánticas, el contenido debe ser claro y fácil de entender. Indente el código, siga buenas prácticas de codificación.

Pilares

Seleccione una paleta de colores, tipografías, íconos, imágenes, que se vean bien y que se complementen. Qué esten en armonía, use las herramientas vistas en clase.
